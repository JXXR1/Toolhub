学習指導案テンプレート

フォームから構造化された学習指導案を生成。標準セクション（目標／教材／指導過程／評価／振り返り）を網羅。Markdown と印刷可能な HTML 出力。

Header
Learning objectives (Bloom's level per objective)
Materials
Instructional steps
Assessment
Differentiation
Reflection (filled in after the lesson) 
上に入力すると結果が表示されます。

用途

学習指導案は「裏側の仕事」です。良い指導案があれば、急な代行の先生もパニックにならずに授業に入れますし、教科主任はチームの動きを把握でき、自分自身も終業後に率直に振り返れます。しかし毎回ゼロから書くのは負担です。構造は毎回ほぼ同じで、変わるのは内容だけ。本ツールは構造をフォームとして提供します。教科、学年、所要時間、目標（Bloom 分類タグ付き）、教材、指導過程、評価、個に応じた指導、振り返り。出力は整った Markdown と印刷可能な HTML プレビューです。

使い方

  • テンプレートを選ぶ。 多くは「標準」で十分。5E は理科の探究サイクルの定番。Gradual Release（「I do / We do / You do」）は直接指導の鉄板フレーム。
  • ヘッダーを埋める。 教科・学年・所要時間は必須。
  • 目標を書く。 生徒が到達する最も高い Bloom レベルでタグ付け。
  • 展開を順序立てる。 時間／段階／教師の動き／生徒の動き。
  • 評価は事前に計画。 形成的は授業中（出口チケットなど）、総括的は授業後（テストなど）。
  • 個別化を明示。 支援の必要な生徒に何を渡す？ 速く終わった生徒は何をする？
  • 振り返りは計画時は空欄。授業後に新鮮なうちに記入。
  • URL に保存。 トグルを ON にすると全状態が URL フラグメントに保存されます。

よくある落とし穴

  • 目標は活動ではない。 「グループで作業する」は活動。「独立変数を特定する」は目標。
  • Bloom はガイドであり、登る義務はない。 鋭い「記憶」レベル目標は曖昧な「評価」レベル目標に勝る。
  • 時間の合計を確認。 50 分授業に 70 分相当の段階を入れたら崩れます。
  • 教師 ≠ 生徒。 生徒欄に 40 分「聞く」と書いてあるなら計画が壊れています。
  • 形成的評価は軽く。 出口チケットやハンドサイン。15 分かかるなら総括的に化けています。
  • テンプレートは出発点。 既定値は意図的に汎用です。遠慮なく書き換えてください。
