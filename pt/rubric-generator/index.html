Gerador de Rúbricas

Gere rúbricas prontas para imprimir (markdown + preview HTML + vista de impressão) a partir de critérios e níveis de desempenho. Feito para professores planejarem avaliações.

Digite uma entrada acima para ver o resultado.

Para que serve?

Uma rúbrica é a coisa mais útil que você pode entregar a um aluno antes do "enviar". Mostra a ele como é um trabalho excelente antes de escrever uma palavra — e mostra a você, que corrige, o que está sendo avaliado. Fazer do zero dá trabalho: montar a grade, combinar nomes de níveis, escrever descritores curtos por critério que realmente diferenciem os níveis. Esta ferramenta te dá um editor familiar (critérios como linhas, níveis como colunas), modelos prontos para redação, projeto, apresentação, relatório de laboratório e trabalho em grupo, e gera tanto um preview HTML imprimível quanto markdown limpo pra colar no Classroom ou no AVA.

Como usar

  • Comece por um modelo e renomeie os critérios pra sua disciplina. Os descritores dos modelos são propositalmente genéricos.
  • Ajuste os níveis. Quatro níveis (Excelente / Proficiente / Em desenvolvimento / Inicial) é o default porque força um julgamento real; três puxa pro "todo mundo no meio" e cinco vira ruído.
  • Escreva os descritores concretos o bastante pra o aluno saber em qual nível está sem você do lado. "Tese clara" é mais fraco que "Tese original mantida em todos os parágrafos do desenvolvimento".
  • Imprima direto do navegador (a vista de impressão esconde o editor). Ou copie o markdown pro AVA.

Cuidados comuns

  • Não use as mesmas palavras em cada nível. "Excelente / Boa / Alguma / Pouca organização" não diz nada. Descreva como cada nível aparece no trabalho.
  • Pontos são pesos, não a nota. 4 níveis × 5 critérios = 5–20 bruto; como vira nota é decisão separada.
  • Evite contagem dupla. Se "argumento" e "análise" se sobrepõem, você pune a mesma fraqueza duas vezes.
  • Células curtas. Descritor comprido parece minucioso mas não é lido. Máximo duas frases por célula.
  • Entregue antes do trabalho começar. Rúbrica revelada depois é só justificativa; usada antes, molda o trabalho.
  • Markdown não preserva quebras de linha. A saída markdown troca quebras por espaços pra não quebrar a tabela. O preview HTML mantém suas quebras.
