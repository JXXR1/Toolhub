¿Para qué sirve?


Esta herramienta lee una imagen, muestrea unos pocos miles de píxeles y los agrupa en 3–10 buckets de colores dominantes con un algoritmo de median-cut implementado a mano. El resultado es un pequeño conjunto de colores hex representativos que puedes usar como semilla de un design system, base de un moodboard o forma rápida de levantar los colores de marca de un logo ajeno. Todo corre en tu navegador; la imagen nunca se sube.



Cuándo usarlo


    
  
  • Sacar los colores dominantes de un logo para construir un tema a juego.
    • 
  
  • Sembrar una config de Tailwind o un bloque de CSS custom-properties desde una imagen hero.
    • 
  
  • Construir una paleta de moodboard a partir de una foto o pieza de arte.
    • 
  
  • Verificar que la imagen hero del diseño y el color de acento de la página concuerden.
    • 




Cómo funciona median-cut


    
  
  1. Reduce la imagen para que el número de píxeles sea manejable.
    2. 
  
  2. Encuentra el eje más largo de la nube de píxeles (rango R, G o B).
    3. 
  
  3. Ordena por ese eje, parte en la mediana.
    4. 
  
  4. Recurre en cada mitad hasta tener el número de buckets pedido.
    5. 
  
  5. Devuelve el color promedio de cada bucket.
    6. 




Errores comunes


    
  
  • Las zonas transparentes se omiten. Píxeles con alfa < 50% no aportan a la paleta — útil para logos sobre fondos transparentes.
    • 
  
  • Fotos con un color dominante (p. ej. una playa con mucho cielo azul) te darán varios tonos de ese color en lugar de una paleta variada. Recorta a una región con más colores si quieres rango.
    • 
  
  • Median-cut es rápido pero no perceptualmente perfecto. Para calidad de teoría del color (complementarios, análogos) seguirás necesitando un diseñador — el algoritmo elige lo más frecuente, no lo más estético.
    • 
  
  • El slider cuenta buckets, no colores "importantes". Pedir 10 desde un logo de 2 colores te dará 10 — solo que serán casi duplicados.
