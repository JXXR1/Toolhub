Markdown से HTML

Markdown को साफ़ HTML में बदलें live preview के साथ। Headings, lists, code, tables, images, और links का समर्थन करता है।

परिणाम देखने के लिए ऊपर इनपुट डालें।

यह किसके लिए है?

Markdown ग्रह पर सबसे अधिक लिखा जाने वाला authoring format है — README files, blog posts, GitHub issues, chat messages, doc sites. HTML वह है जो browsers render करते हैं। यह टूल Markdown को साफ़ HTML में live preview के साथ बदलता है ताकि आप यह देख सकें कि render किया गया output कैसा दिखेगा इससे पहले कि आप इसे CMS में paste करें, static page generate करें, या email template के हिस्से के रूप में भेजें।

समर्थित syntax

  • Headings # से ###### तक; bold, italic, strike
  • Inline `code` और fenced ```lang blocks
  • Bullet / numbered / nested lists
  • Links [text](url) और images ![alt](url)
  • Blockquotes, alignment वाले pipe tables, horizontal rules

आम गलतियाँ

  • यह एक तेज़ in-browser parser है, spec suite नहीं। CommonMark और GFM में edge cases (nested emphasis, link reference definitions, autolink expansion) हैं जो implementations के बीच भिन्न होते हैं। सख्त conformance के लिए, build step में marked, markdown-it, या remark का उपयोग करें।
  • Markdown के अंदर embedded HTML अधिकतर as-is pass हो जाता है, लेकिन कुछ implementations इसे sanitise करते हैं। सुरक्षा के लिए इस पर भरोसा न करें; untrusted markdown को untrusted HTML के रूप में treat करें।
  • Tables बनाम alignment। Pipe tables को एक separator row (|---|) चाहिए और centre/left/right alignment के लिए :---: / :--- / ---: का उपयोग करते हैं। Separator भूलना सबसे आम कारण है कि एक "table" एक paragraph के रूप में render होती है।
  • Smart punctuation। कुछ renderers -- को en-dashes में और straight quotes को curly में बदल देते हैं। यह टूल नहीं करता — यदि आपको ज़रूरत है तो typography pass से गुज़रें।
  • Round-tripping lossless नहीं है। Markdown → HTML → Markdown heading style, list spacing, और link form को normalise करेगा। Semantics बच जाती हैं; exact bytes नहीं।
