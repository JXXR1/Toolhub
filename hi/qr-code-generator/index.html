QR Code Generator

किसी भी text, URL, Wi-Fi, या vCard के लिए QR code बनाएं। कस्टम size, रंग, और error correction। PNG या SVG में export करें।

यह किसके लिए है?

QR code छोटे URLs होते हैं जिन्हें एक phone camera सेकंड के अंश में पढ़ सकता है। एक URL, एक Wi-Fi network, एक email draft, एक phone number, या कोई भी छोटा text encode करें — इसे एक poster, business card, या menu पर print करें, और किसी भी phone वाला बिना टाइप किए इस पर action ले सकता है। यह generator पूरी तरह आपके browser में open-source qrcode library का इस्तेमाल करके चलता है। आपका input text कभी page से बाहर नहीं जाता।

कब इस्तेमाल करें

  • एक slide, poster, business card, या packaging insert पर URL share करना।
  • मेहमानों को password टाइप किए बिना अपने Wi-Fi से जोड़ना (Wi-Fi preset का इस्तेमाल करें)।
  • भौतिक स्थानों में signage पर "scan to pay / book / order" लगाना।
  • प्राप्तकर्ता के phone पर तत्काल contact-card import के लिए एक vCard encode करना।

आम गलतियाँ

  • Error correction density से trade-off करता है। उच्च स्तर (Q, H) खरोंच और overlaid logos से बचते हैं; निम्न स्तर (L, M) code को छोटा और scan करने में तेज़ रखते हैं। यदि आप logo overlay करने की योजना बना रहे हैं तो H का इस्तेमाल करें, अन्यथा M समझदार default है।
  • Contrast मायने रखता है। कम foreground/background contrast वाले stylised QR codes अक्सर सुंदर दिखते हैं पर खराब रोशनी में खराब scan होते हैं। ship करने से पहले printed/screen output से scan करके test करें।
  • लंबी content denser code को मजबूर करती है। यदि आपको 200-character URL encode करना है, तो परिणामी code dense है और दूर से scan करना मुश्किल। यदि size मायने रखता है तो पहले redirect के माध्यम से छोटा करें।
  • SVG vs PNG। SVG बिना quality loss के scale होता है — print या बड़े displays के लिए सबसे अच्छा। PNG सार्वभौमिक रूप से स्वीकृत है पर scale up करने पर pixelate होता है; उस size पर export करें जिस पर आप इस्तेमाल करेंगे।
  • Wi-Fi preset format। WIFI:T:WPA;S:<SSID>;P:<password>;; — आधुनिक iOS/Android scan करते हैं और automatically join करते हैं; बहुत पुराने phones शायद support न करें।
