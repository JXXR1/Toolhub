Untuk apa ini?

Menghitung kata dan karakter secara manual itu membosankan dan rawan error, padahal angkanya selalu penting: batas tweet, segmen SMS, jumlah kata esai, meta description SEO, panjang submission majalah. Tool ini memberi hitungan live saat kamu mengetik — kata, karakter (dengan dan tanpa spasi), kalimat, paragraf, baris — plus estimasi waktu baca dan waktu bicara, dan ringkasan cepat lima kata paling sering muncul untuk frekuensi kata.

Kapan digunakan

  • Mengejar batas 280 karakter X/Twitter, SMS 160 karakter, meta description SEO 155 karakter, atau intro LinkedIn 100 kata.
  • Mem-budget esai, blog post, abstract, atau aplikasi hibah terhadap batas keras.
  • Memperkirakan berapa lama script akan dibaca keras-keras (podcast, presentasi, voice-over).
  • Mendeteksi kata yang terlalu sering dipakai dengan melihat daftar "paling sering" sebelum submit.
  • Mengecek apakah terjemahan datang kira-kira sepanjang sumber.

Arti setiap angka

  • Kata — kelompok karakter non-whitespace yang dipisah whitespace. "Twenty-one" dihitung satu kata; "twenty one" dihitung dua.
  • Karakter vs karakter (tanpa spasi) — keduanya menghitung Unicode code point, bukan byte. Sebuah emoji di sini bisa 1–2 "karakter" tapi lebih banyak byte saat disimpan.
  • Kalimat — segmen yang diakhiri ., !, atau ? (atau akhir teks). Bersifat heuristik, lihat common gotchas.
  • Paragraf — dipisah baris kosong.
  • Waktu baca mengasumsikan 250 wpm (baca diam dewasa).
  • Waktu bicara mengasumsikan 130 wpm (tempo bicara tipikal; news anchor lebih cepat, audiobook lebih lambat).

Kesalahan umum

  • Deteksi kalimat itu naif. "Mr.", "U.S.", "e.g.", "3.14", dan ellipsis bisa menggelembungkan hitungan kalimat. Angkanya adalah estimasi berguna, bukan jaminan.
  • Twitter/X menghitung code point, bukan karakter. Emoji bendera (🇮🇩) adalah 2 code point tapi di-render sebagai satu simbol — Twitter memperlakukannya sebagai 2 karakter. Tool ini mencocokkan itu.
  • Batas karakter SMS tergantung encoding. ASCII murni muat 160 karakter per segmen; tambahkan satu karakter non-GSM (em dash, smart quote, huruf beraksen) dan seluruh pesan beralih ke UCS-2 dan batas turun ke 70. Tool melaporkan batas GSM; cek perilaku provider untuk biaya sebenarnya.
  • Jumlah kata berbeda antar tool. Word, Google Docs, dan sistem submission majalah bisa beda beberapa persen — mereka menangani tanda hubung, em dash, dan angka secara berbeda. Jika batas keras penting, hitung di tool yang sama dengan yang dipakai gatekeeper.
  • "Paling sering" tidak memfilter stop word. "the" dan "a" hampir selalu di atas. Lihat input yang lebih panjang untuk sinyal nyata.
  • Estimasi waktu baca itu personal. 250 wpm adalah median; konten teknis lebih lambat, fiksi lebih cepat. Perlakukan angka sebagai panduan planning, bukan prediksi.
