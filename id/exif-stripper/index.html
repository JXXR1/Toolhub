Apa itu EXIF dan kenapa peduli?

Setiap foto yang kamu ambil dengan ponsel atau kamera membawa blok data tersembunyi — metadata EXIF — yang tertanam di file di samping piksel. Kebanyakan orang tidak pernah melihatnya. Tapi ada di sana: koordinat GPS tempat foto diambil (sering akurat sampai beberapa meter), detik tepat di jam ponselmu saat rana terlepas, merek dan model perangkat, lensa, kadang namamu dari kolom "owner" kamera. Upload JPG itu ke forum, kirim lewat WhatsApp, post ke blog pribadi, dan data itu ikut — kecuali platform-nya kebetulan membersihkannya.

Tool ini membaca EXIF langsung di browser-mu (tanpa upload — JavaScript di halaman mem-parse byte-nya), menunjukkan apa yang ada, dan menawarkan unduhan satu klik foto yang sama tanpa metadata. File asli tidak pernah meninggalkan perangkatmu.

Apa yang sebenarnya terekspos

  • Koordinat GPS. Kalau location services menyala saat memotret, lintang/bujur tertanam. Enam digit desimal ≈ 11 cm presisi — cukup untuk mengidentifikasi alamat rumah.
  • Tanggal dan waktu. Sampai detik, di zona waktu lokal kamera.
  • Model kamera + serial. Beberapa kamera menanam serial unik yang menghubungkan banyak foto ke body fisik yang sama.
  • Lensa, bukaan, ISO, focal length. Berguna untuk fotografer; tidak berbahaya.
  • Software editing. "Photoshop CC 2023" mengungkap bahwa gambar telah diproses, kadang dengan identitas editor.
  • Field owner / artist / copyright. Beberapa kamera memungkinkan default nama. Ia ikut setiap foto.

Kapan harus strip EXIF

  • Sebelum posting foto rumah, tempat kerja, atau lokasi yang tidak ingin tertanda di peta.
  • Sebelum publik berbagi foto "di sini aku ambil" saat kamu lebih suka tidak mengungkap di mana tinggal.
  • Saat mengirim foto ke jurnalis, pengacara, support group, atau siapa pun dalam konteks sensitif.
  • Workflow whistleblower / perlindungan sumber — strip metadata sebelum upload bukti gambar.
  • Saat mendaftarkan di marketplace (OLX, Facebook Marketplace) — foto sofa-mu bisa mengungkap alamat.

Apa yang tool ini lakukan dan tidak lakukan

  • Lakukan: Membaca EXIF dari file JPG. Menampilkan GPS, kamera, timestamp, lensa, software. Memancarkan ulang JPG tanpa segmen APP1 (EXIF), APP13 (Photoshop IRB / IPTC), atau APP14 (Adobe).
  • Lakukan: Berjalan sepenuhnya di browser-mu. Tidak ada yang di-upload. Tidak pernah ada log server file ini.
  • Tidak lakukan: Menangani EXIF di format RAW (CR2, NEF, ARW) — itu butuh tool desktop. PNG dan WebP jarang membawa EXIF.
  • Tidak lakukan: Menghapus metadata yang terbakar di piksel (mis. timestamp terlihat di gambar, atau watermark).
  • Tidak lakukan: Meng-encode ulang JPG. Data piksel byte-identik; hanya segmen metadata yang dihapus, jadi kualitas tidak berubah.

Kesalahan umum

  • Jejaring sosial berbeda-beda. Facebook dan Twitter strip EXIF saat upload; Instagram secara historis juga. Tapi situs image hosting, forum, dan blog pribadi sering tidak. Perlakukan setiap platform sebagai untrusted sampai kamu menguji.
  • Screenshot biasanya tidak punya EXIF. Tool akan memberi tahu kalau sudah bersih.
  • Cloud sync menyimpan aslinya. Strip EXIF dari JPG yang kamu unduh tidak mengubah salinan yang masih di iCloud / Google Photos.
  • Thumbnail bisa bertahan. Beberapa software menyimpan preview JPEG kecil di dalam blok EXIF; tool ini membuang segmen EXIF utuh, jadi preview itu juga hilang.
