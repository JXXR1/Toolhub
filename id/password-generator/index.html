Password Generator

Password acak yang kuat atau passphrase yang mudah diingat. Dihasilkan lokal — tidak pernah dikirim ke mana pun.

Masukkan input di atas untuk melihat hasilnya.

Untuk apa ini?

Password yang baik adalah password yang tidak bisa ditebak attacker dan tidak perlu kamu ingat (karena tersimpan di password manager). Generator ini menghasilkan password random yang kuat atau passphrase yang mudah diingat sepenuhnya di browser kamu, menggunakan crypto.getRandomValues — sumber random kriptografis aman yang sama yang dipakai TLS. Tidak ada yang dikirim; password tidak pernah meninggalkan device kamu.

Kapan digunakan

  • Membuat password unik untuk setiap akun baru yang masuk ke password manager.
  • Menghasilkan master password atau passphrase recovery yang akan kamu hafalkan — mode passphrase lebih mudah diketik dan diingat.
  • Menghasilkan secret non-manusia untuk CI variable, API token, atau jaringan Wi-Fi.
  • Hasilkan password secara batch untuk batch user baru (atur count sampai 50).

Karakter random vs passphrase

  • Karakter random — entropy paling banyak per panjang. 20 karakter campuran ≈ 130 bit. Cocok untuk hal yang kamu paste, bukan ketik.
  • Passphrase — lebih mudah diketik dan diingat. Empat kata ≈ 40 bit, enam kata ≈ 60 bit. Cocok untuk master password, unlock device, dan apa pun yang akan kamu ketik manual secara rutin.
  • "Exclude ambiguous" membuang 0/O/1/l/I agar lebih aman dibaca dari layar atau catatan tulisan tangan.

Berapa banyak entropy yang dibutuhkan?

  • ≥ 60 bit — cukup untuk akun bernilai rendah
  • ≥ 80 bit — bagus untuk sebagian besar akun
  • ≥ 100 bit — bernilai tinggi (finansial, master password, root credential)

Kesalahan umum

  • Jangan menggunakan ulang password. Upgrade keamanan terbesar yang bisa kamu lakukan adalah satu password unik per situs, disimpan di manager. Kekuatan generator sia-sia jika password yang sama dipakai di lima situs.
  • Jangan menulis password yang dihasilkan tanpa proteksi. Gunakan password manager (1Password, Bitwarden, KeePass) — bukan app Notes, bukan file teks, bukan draft email.
  • Panjang > kompleks. Password 24 karakter yang hanya pakai huruf kecil punya entropy lebih banyak dibanding 10 karakter dengan setiap kelas simbol. Panjang yang menang.
  • Aturan spesifik situs bisa merusak copy-paste. Beberapa situs melarang simbol tertentu atau membatasi panjang di 16. Menyebalkan tapi nyata — hasilkan, lalu trim/swap agar sesuai jika perlu (dan simpan password aktual yang dipakai di manager kamu).
