Wofür ist das?

Dieses Tool ist die Umkehr des Bild-zu-Base64-Encoders: Es nimmt eine data:-URI (oder Roh-Base64) und dekodiert sie zurück zu einem anzeigbaren, herunterladbaren Bild. Das Format wird automatisch aus dem MIME-Typ der URI erkannt — oder, wenn du nur Roh-Base64 lieferst, anhand der ersten Bytes (PNG-Signatur, JPEG-SOI-Marker, GIF8-Header, RIFF…WEBP, SVG-Markup) bestimmt. Alles läuft im Browser über atob und ein <img> mit Data URI.

Wann verwenden

  • Das Originalbild zurückgewinnen, wenn du nur ein Data URI aus CSS, HTML oder einer Markdown-Datei kopiert hast.
  • Einen Base64-String aus einer API-Antwort oder einem JWT-ähnlichen Token inspizieren.
  • Ein Base64-Bild debuggen, das woanders nicht rendert — durch dieses Tool drehen, um den Payload zu bestätigen.
  • Ein eingebettetes Asset aus einer Single-File-HTML-Demo oder Self-Contained-E-Mail extrahieren.

Häufige Stolpersteine

  • Beide Formen werden akzeptiert. Komplettes data:image/png;base64,iVBORw0… oder nur Roh-iVBORw0… — aber bei nicht erkennbarem Format wird der Download als .bin getaggt.
  • URL-safe Base64 ist nicht ganz dasselbe. Standard-Base64 nutzt + /, URL-safe nutzt - _. JWT-Payloads sind URL-safe — vor dem Einfügen umwandeln.
  • Whitespace im Payload ist okay. Der Decoder entfernt Zeilenumbrüche und Leerzeichen, mehrzeilige Strings funktionieren.
  • Browser rendern kein kaputtes Bild. Wenn die Vorschau leer bleibt, aber das Decoding klappt, ist der Payload zwar gültiges Base64, aber wahrscheinlich kein echtes Bild (oder ein nicht unterstütztes Format).
